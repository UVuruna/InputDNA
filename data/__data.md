# data/

Runtime data directory. Contains the SQLite database generated by the recorder
during active recording sessions. This folder is empty in the repository —
the database is created automatically on first run.

<a id="folder-structure"></a>

## Folder Structure

```
📁 data/
  📝 __data.md
  🗄️ movements.db          ← created at runtime (gitignored)
  📄 movements.db-wal      ← SQLite WAL file (gitignored)
  📄 movements.db-shm      ← SQLite shared memory (gitignored)
```

<a id="database"></a>

## `movements.db` — Main Recording Database

SQLite database (WAL mode) containing ALL recorded human input data.
Created by `database/schema.py` on first launch via `main.py`.

**Database size estimate:** ~150-200 MB per month of typical desktop use.

<a id="wal-files"></a>

## WAL Files

During active recording, SQLite creates sidecar files:

| File | Purpose |
|------|---------|
| `movements.db-wal` | Write-Ahead Log — buffered writes before merge |
| `movements.db-shm` | Shared memory — coordination for WAL access |

> **Warning:** Do NOT delete WAL/SHM files while the recorder is running.
> They get merged back into the main database file on clean shutdown.

<a id="backup"></a>

## Backup

The database is the single most valuable file in this project — it contains
all your recorded behavior data that cannot be re-collected without repeating
the recording sessions. Back it up periodically.

<a id="gitignore"></a>

## .gitignore

All `*.db`, `*.db-wal`, and `*.db-shm` files in this directory are gitignored.
The database is personal data generated per-user and should never be committed
to version control.
